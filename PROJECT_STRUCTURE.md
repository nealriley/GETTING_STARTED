## ğŸ¨ Design Principles

1. **Empathetic, not condescending**
   - Matter-of-fact tone about technical concepts
   - Never explicitly call out intimidation

2. **Show, don't just tell**
   - Concrete examples over abstractions
   - Live demonstrations in animation
   - Before/after comparisons

3. **Playful but precise**
   - Memorable metaphors (chef, vending machine, factory)
   - Definitions with personality
   - Accuracy never sacrificed for style

4. **Structure as content**
   - Guide demonstrates its own principles
   - Meta-commentary throughout
   - "Getting Started" as documentation example
