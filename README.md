## Content Principles

1. **Empathetic, not condescending** - Never call out intimidation, just treat concepts as matter-of-fact
2. **Show, don't just tell** - Concrete examples over abstractions
3. **Playful but precise** - Memorable metaphors without sacrificing accuracy
4. **Structure as content** - The guide itself demonstrates good documentation

## Target Audience

**Primary:** "The Technical Adjacent"
- UX researchers, product managers, designers, data analysts
- Work WITH technical systems daily
- Hear terms like "CLI," "MCPs," "prompt engineering" but don't use them
- Gap between knowing WHAT and knowing HOW
